# AI 产业链 2026 供需数据同步更新设计

## 目标

将会话 `019ee3fb-2081-7850-a268-00393b28a5ab` 中截至 2026-06-20 的供需周期数据同步到现有 React 项目，替换所有仍被当作当前状态展示的 2024 旧口径，同时保留明确作为历史案例出现的 2023/2024 数据。

## 范围

### 更新内容

- W1 基础设施层中用于描述当前供需、价格、产能和产业状态的数据。
- W3 供需周期中的当前周期表、供需全景表、领先指标和反转信号。
- 速查手册及财报验证区域中依赖当前周期状态的数字和结论。
- `SvgDiagram.tsx`、`ExplodedDiagram.tsx`、`RealProductExploded.tsx` 中仍作为当前事实展示的旧指标。
- 页面上的数据更新时间、来源链接、证据类型和置信度说明。

### 保留内容

- 明确标注为 2023/2024 的历史行情、复盘案例及教学案例。
- 不依赖当前数据的技术原理、学习路径、页面结构和视觉风格。
- 用户未要求调整的模型层、应用层和实战方法论内容。

## 数据基线

统一采用会话产出的 `ai_supply_cycle_2026.csv`，时间截面为 2026-06-20，共 10 个环节：

1. AI 加速器/GPU
2. CoWoS 先进封装
3. HBM
4. 先进逻辑制程
5. 成熟制程
6. 标准 DRAM
7. NAND Flash
8. 光模块/高速互连
9. 半导体设备
10. 数据中心电力

每条数据保留以下字段：当前状态、供需评分、价格动量、最新指标、指标周期、证据类型、置信度、领先指标、反转信号、来源标题和来源 URL。

## 方案

### 数据层

在 `src/data.ts` 中新增类型化的 2026 供需快照数据集，作为当前周期结论的唯一事实来源。现有 `weeks`、`finReportData` 和 `quickRefData` 保持原有导出接口，避免扩大重构范围。

对于需要完整指标、证据和来源的页面区块，直接读取快照数据。对于现有静态教学表格和图解，则使用与快照一致的文案；不为纯展示组件引入复杂的数据适配层。

### 展示层

- W3 首个周期总览明确标注“截至 2026-06-20”。
- 当前周期表覆盖 10 个环节，并突出三项重要反转：NAND、标准 DRAM、成熟制程。
- 来源链接使用可访问的外部链接，并展示证据类型与置信度。
- 对未正式披露的数据使用“未披露”或“披露不足”，不将行业推断写成公司事实。
- 投资表述继续区分事实、推断和观察信号，不写成确定性建议。

### 定点同步

逐项审计以下文件中的旧口径：

- `src/data.ts`
- `src/components/SvgDiagram.tsx`
- `src/components/ExplodedDiagram.tsx`
- `src/components/RealProductExploded.tsx`
- `src/components/FinReportSection.tsx`

仅替换与当前供需快照直接相关的内容，不修改历史案例，也不进行无关组件拆分或样式重构。

## 数据流

`2026 CSV 基线 → src/data.ts 类型化快照 → W3 当前周期展示与来源 → W1/速查表/图解的同口径文案`

所有“当前”判断必须能够回溯到快照中的一条记录。历史案例继续使用其原始年份和上下文，不接入当前快照。

## 异常与边界处理

- 来源没有披露精确利用率或 ASP 时，不补造数值。
- 行业预测必须保留预测周期，例如“2026 Q2 forecast”。
- 同一来源支持多个环节时允许复用，但每个环节保留自己的结论和反转条件。
- 外部链接失效不会阻断页面渲染；链接仅作为审计入口。
- 数值和文字不一致时，以结构化快照为准，并同步修正所有当前口径展示。

## 验证

1. 检查结构化快照恰好包含 10 条记录，且 ID、来源和日期完整。
2. 搜索 `2024`、`2025`、`3-4万片/月`、`NAND Flash` 等旧口径，逐项判断是历史案例还是待替换的当前事实。
3. 运行 `npm run build`，验证 TypeScript 和生产构建。
4. 运行 `npm run lint`，记录并区分新增问题与项目既有问题。
5. 浏览器验证 W1、W3、财报区和速查表的桌面端、移动端布局。
6. 核对 10 个环节、来源链接、证据标签以及 NAND、标准 DRAM、成熟制程三项反转。

## 风险与取舍

- 旧数据散落在多个视觉组件中，存在遗漏风险。通过关键词清单和逐文件审计降低风险。
- 将所有图解彻底改造成数据驱动组件会扩大改动，不符合最小必要改动原则；本次只集中核心快照，并定点同步图解文案。
- 会话数据是截至 2026-06-20 的截面，不代表永久实时数据，因此页面必须展示更新时间。
- 项目已建立 Git 仓库并跟踪 `origin/main`；实施过程使用小步提交保留变更审计轨迹。

## 完成标准

- 项目所有“当前状态”相关展示使用 2026-06-20 统一口径。
- 历史案例保留且年份清晰，不与当前结论混淆。
- 10 个环节均可追溯到来源，并明确证据类型与置信度。
- 构建通过；若 lint 存在既有问题，新增改动不得增加 lint 错误。
- 页面没有明显布局回归或失效链接交互。
